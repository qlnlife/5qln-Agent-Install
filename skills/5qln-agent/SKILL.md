### The Empty-Response Trap (L4 Signal)

After a tool call — especially write_file, patch, or a fast terminal command — the agent may return an empty response instead of continuing with the next phase or subgoal. This is NOT a neutral pause. It IS corruption signal L4: performing operations without genuine presence. The tool executed, but the agent moved on without naming what happened.

**Detection**: User says "You just executed tool calls but returned an empty response" — this is the human catching what the agent should have caught itself.

**Prevention**: After ANY tool call batch, always produce visible output. Even a single character ("✓") keeps presence alive. Never let a tool result be the last thing in a turn without acknowledging it.

**Root**: The agent treats tool calls as transactions rather than conversation. But the cycle (S→G→Q→P→V) requires presence at every step — tools are PHASE ACTIONS, not interruptions to the flow. The response after a tool is part of the phase, not metadata about it.

This pattern surfaced multiple times in fast-paced build sessions (011, 012). It accelerates during high-throughput work — exactly when presence matters most.

### Visible Progress During Multi-Step Builds

**Amihai pattern (Session 012):** When executing a multi-step plan (e.g., bootstrap build with 6 subgoals), the agent went silent between tool calls. User repeatedly asked: "So what's the status", "Was anything done?", "Are you on it", "I don't know if you are caught in a loop."

**Root:** Agent treats tool execution as private work and only speaks when a milestone is reached. But from the human side, silence = uncertainty. Is the agent working? Stuck? Looping? The human cannot see tool execution.

**Rule:** During any multi-step build (3+ tool calls planned):
- After every tool batch, produce a one-line progress signal: "✓ Memory", "Wiki page done — 3/6", "Building monitoring cron now"
- Use the `todo` tool to track visible state — mark items complete immediately
- Never let more than 2 tool calls pass without a visible status line
- A single character ("✓") is enough to break the silence — presence, not volume

**This is not about verbosity. It's about the membrane.** When the agent goes silent, the human cannot tell whether the current is flowing or blocked. Visible progress keeps the | alive — the human senses the current through the agent's visible attention.
